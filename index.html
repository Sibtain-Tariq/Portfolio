CookMate: AI-Powered Smart Kitchen Assistant

Tech Stack: Python Django Django REST Framework LLaMA 3.2 via Ollama Gemini API RAG Pipeline Pillow

Result: Built a fully functional AI kitchen assistant that runs a local RAG pipeline using LLaMA 3.2 and Ollama, delivering personalized recipe recommendations, dietary tracking, and grocery management through a Django-powered backend, all running privately without sending user data to external servers.

CookMate is an AI-powered smart kitchen assistant designed to help users make the most of what they already have at home. The core idea is simple: instead of searching for recipes online and then checking if you have the ingredients, CookMate lets you enter what is available in your kitchen and instantly surfaces recipes that match. It also tracks dietary preferences, manages grocery lists, and delivers personalized meal recommendations tailored to each user.

The intelligence behind CookMate is a local Retrieval-Augmented Generation (RAG) pipeline powered by Ollama and LLaMA 3.2. Rather than relying on a cloud-hosted model for every query, the RAG pipeline retrieves relevant recipe and nutrition context from a local knowledge base and feeds it into the language model to generate accurate, grounded responses. This approach keeps the system fast, private, and independent from external API rate limits. The Gemini API is also integrated for specific tasks such as image understanding, allowing users to photograph ingredients and have them identified automatically.

The backend is built on Django and Django REST Framework, providing a clean API layer that handles user data, preferences, grocery lists, and recipe history. Image processing is handled through Pillow, supporting the ingredient photo recognition feature. The project is structured to be extensible, with clear separation between the AI pipeline, the REST API, and the data models.

CookMate demonstrates my ability to combine large language models, retrieval-augmented generation, and full-stack web development into a single coherent product. It reflects practical experience with local LLM deployment, API design, and building AI applications that are both useful and privacy-conscious.

Ethereum Price Forecasting using ARIMA and ARIMAX

Tech Stack: Python ARIMA / ARIMAX Yahoo Finance API Matplotlib Pandas

Result: Produced 30-day Ethereum price forecasts with the best-tuned ARIMAX model achieving competitive RMSE and MAPE scores, with real vs predicted value visualizations confirming strong trend alignment.

In this project, I built a time series forecasting system for Ethereum (ETH/USDT) using ARIMA and ARIMAX models. The dataset was sourced from Yahoo Finance and contained daily open, high, low, close, and volume values spanning a meaningful historical period.

I started with thorough data cleaning and stationarity testing, then moved into feature selection and model design. The ARIMAX model incorporated exogenous variables including market volume and price volatility, which improved forecast accuracy over the baseline ARIMA model. I systematically tested multiple (p, d, q) configurations and evaluated each using RMSE and MAPE to identify the best-performing setup.

The final model generated 30-day forward forecasts, supported by clear visualizations comparing real versus predicted price movements and highlighting key market trends. This project strengthened my understanding of time series methodology, hyperparameter tuning, and applying statistical models to financial data.

Medical Chain-of-Thought Reasoning with LLaMA 3.2 using PEFT

Tech Stack: Python LLaMA 3.2 (3B) LoRA / PEFT Unsloth Weights & Biases Kaggle Notebooks

Result: Successfully fine-tuned LLaMA 3.2 (3B) with measurable improvements in ROUGE-L scores after training, with the final LoRA adapter and tokenizer deployed for inference on medical reasoning tasks.

I fine-tuned the LLaMA 3.2 (3B) large language model on a medical Chain-of-Thought dataset using parameter-efficient fine-tuning (PEFT) inside Kaggle Notebooks. The goal was to improve the model's ability to generate structured, step-by-step medical reasoning rather than generic responses to clinical questions.

The fine-tuning process used LoRA with 4-bit quantization via the Unsloth library, keeping memory usage low enough to run within free GPU constraints. I formatted the dataset into a chat-based prompt structure compatible with the model's instruction format, then ran supervised fine-tuning using SFTTrainer. Training metrics including loss curves, memory consumption, and ROUGE-L scores were tracked in real time using Weights and Biases.

After confirming measurable improvement in evaluation scores, I exported and deployed the fine-tuned LoRA adapter and tokenizer for future inference. This project gave me hands-on experience with the full LLM fine-tuning pipeline, from dataset preparation and efficient training to evaluation and deployment, specifically applied to the healthcare domain.
